Design: Commands & Tool Display

Slash commands and tool call rendering

Slash Commands (Before → After)

Before: Complex Command System
/quit — exit
/model — switch model (also Ctrl+L picker)
/session — show session info
/resume — restore session (also Ctrl+R picker)
/tree — tree navigation (also Ctrl+T picker)
/compact — manual compaction (also Ctrl+K)
/help — help text
/status — status detail (also Ctrl+Y)
+ extension commands via pool
+ fuzzy matching via FuzzyCompleter
+ @ path completion
After: Minimal Commands
/quit
Exit the session
/model [name]
Switch model (list if no arg)
/resume [id]
Resume session (list if no arg)
/help
Show available commands
/compact
Manually trigger compaction
5 commands total. Simple WordCompleter on /.
No pickers. No @ path. No extension commands.

Tool Call Display

The key feature to preserve — showing tool execution progress during agent runs.

Tool Call Rendering in StreamView
Assistant:
Let me read the file and check the function.
⠋ read_file src/main.py running...
✓ edit_file src/utils.py 0.3s
✗ bash pytest tests/ 1.2s
Implementation: Each tool call is a simple dataclass with name, args_summary, status (running/done/error), duration. Rendered as Rich.Text with colored prefix. No diff view, no expandable details, no toggle shortcuts.
